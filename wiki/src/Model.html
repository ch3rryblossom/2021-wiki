Introduction

Mathematical modelling has become an integral part of almost all projects dealing with metabolic engineering. Information starting from the dynamics of our system, to what genetic modifications can further our project can be obtained from the right mathematical models. These models help as starting points for the wet-lab experiments as they give a rough idea of where to target your attention.

SynBactory being heavily dependent on organism’s growths, behaviours and interactions, along with metabolic networks and its possible manipulations, is a project that can benefit a lot from mathematical models.

The main objectives were:

  1. Predict growth rates of our individual organisms in pure cultures
  2. Predict growth rates under different stresses
  3. Identify the important interactive metabolites in the co culture
  4. Identify targets to over express or knockout in each organism for improved yields of sucrose and butanol

The core technique of our modelling attempts is a constraint based modelling technique called Flux Balance Analysis, in short FBA. We have used the COBRA package in MATLAB for almost all our models.

Flux Balance Analysis

FBA is a steady state analysis algorithm that is widely used in metabolic predictions and optimisations. FBA assumes the organism to be in a metabolically steady state i.e., there is no accumulation of metabolites within the organism. To perform FBA we require a metabolic model of the organism. A metabolic model of an organism contains information on all (or most) reactions in an organism, the metabolites involved, the genes present in the organism, the Gene-Reaction-Relation, and a calculated biomass production reaction (growth rate reaction) that gives the growth rate when analysed.

The core elements of FBA are:

  1. S Matrix or the stoichiometric matrix - This is a matrix of the order (number of metabolites) * (number of reactions)
  2. The objective function or c - This is a column vector that holds the information to attain the aim of the analysis, for example biomass optimisation
  3. Lower bounds and upper bounds - This is a set of values that help constrain the solution space
  4. Solution flux space - This is a set of fluxes that give the same objective function value

In an organism, it is normal to have the same metabolites participating in multiple reactions, and therefore it is safe to assume the number of reactions will be more than the number of metabolites. This can be compared to a system of linear equations with more equations than variables. Without constraints the solution space is infinite, so we introduce constraints in the form of bounds.


Equations of FBA and their solutions

Constraints:

  • Sv = 0
  • Lower Bounds
  • Upper Bounds

Objective Function:

  • C(transpose)v

Objective:

  • Minimise or Maximise Objective function

Solution:

  • Plausible flux distributions for the above mentioned objective function

If one has perturbed the metabolic network of the organism, then simple maximisation of growth rate does not give results that agree with experiments. MoMA is a widely used algorithm to identify solution sets that agree most with real life values.

Minimisation of Metabolic Adjustments

MoMA is an extension of Flux Balance Analysis that imposes more constraints to make the model as close to reality as possible, albeit it isn’t the best at doing so. The logic of MoMA is very simple - minimise the euclidean distance (L2 norm) between the wild-type flux distribution and the flux distribution of the modified organism in the flux vector space.


The algorithm:

  1. Find the wild-type/base strain flux distribution using FBA for maximized growth rates
  2. Simulate the deletion, over-expression or any stress
  3. Impose an additional constraint that minimizes the L2 norm of the solution and the wild type flux; it would look something like this - ||v_wild - v_mod||^2 is minimised.
  4. Run FBA multiple times to find such a v_mod.

Fascinating thing is this algorithm quite literally translates into the name of the technique.

Here we know what the modifications are that we need to make in order to get the strain we need, but what if we need to find further modifications that would help increase the yield of the product, simple FBA or MoMA will not suffice for this purpose. This is where we start using techniques like FSEOF and OptKnock.

Flux Scanning based on Enforced Objective Flux (FSEOF)

This algorithm was developed to identify gene targets that can be amplified for increased production of selected metabolites or even increased growth rates. We have used this technique to identify the reactions and further genes that we can over-express to increase sucrose production in cyanobacteria and butanol production in E. coli.

To understand FSEOF imagine the whole organism as a big city, the reactions as roads, and the flux through the reactions as the traffic in each road. Now consider evacuating this city as our objective, then the roads which lead to the exits will start carrying more traffic. If we can widen these roads we can in theory get more people out. If evacuating this city was another way of saying - maximise the production of a particular metabolite, then widen the roads can be considered as amplifying the corresponding reactions.


In the form of an algorithm it would look something like this:

  1. Find maximum growth under media conditions using FBA
  2. Find the maximum production of the metabolite using FBA
  3. Constraint the production of the metabolite to some fraction of the maximum metabolite production value
  4. Run FBA for maximum growth rate
  5. Calculate the product of the metabolite production value for a given constraint and the corresponding growth rate value. This would help rank the target later on
  6. Plot how the flux values of each reaction changes as we increase the fractional value of the constraint on the metabolite

Since FBA and all of its extensions work with reactions and not genes we will have to trace the reactions back to its controlling genes and see if those genes are responsible for any other process that could end up killing the organism or hindering its growth if amplified.

OptKnock

OptKnock is an algorithm that find and suggests which set of reactions to remove from a metabolic network to obtain a mutant that will produce a particular target of interest at a higher rate than the wild-type strain. The logic is pretty simple -

  1. You find the set of reactions that you want to get knockouts from
  2. Change their bounds to 0 (this is how you simulate deletion in models) either one by one or in pairs or triplets
  3. Run FBA for maximised growth
  4. Once you have a set of reactions which when deleted increases the production of the target of interest you trace them back to their respective genes
  5. Once you’ve identified the genes you eliminate all the synthetic lethals from the set. Synthetic lethals are essential genes that was required by the organism to live

Once you’ve eliminated all the synthetic lethal you have a set of reactions and their corresponding genes which can be knocked out for enhanced production of the metabolite of interest.
